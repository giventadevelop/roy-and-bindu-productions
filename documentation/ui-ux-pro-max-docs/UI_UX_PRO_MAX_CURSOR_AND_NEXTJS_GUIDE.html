Overview

This guide documents installing UI/UX Pro Max for the Roy and Bindu Productions Next.js workspace: global uipro-cli, project skill files under .cursor/skills/ui-ux-pro-max/, Windows notes, comparison with the frontend-design skill, and Next.js-oriented python commands for design-system generation and persistence.

Global install — npm

Install the CLI once per machine (npm package uipro-cli@2.2.3 as verified at install time).

Setup
🔧 Customize Parameters:
npm install -g uipro-cli
Windows single-line:
npm install -g uipro-cli
Project init — Cursor

From the repository root, run init so files are written to .cursor/skills/.

Setup
🔧 Customize Parameters:
uipro init --ai cursor
Windows single-line:
uipro init --ai cursor
Success output (example): installer reports files under .cursor/skills/ and suggests restarting the AI assistant. After install, the skill includes SKILL.md, scripts/search.py, and data/*.csv.
Windows: PATH and running uipro
⚠️ If uipro is not recognized: the global npm bin folder may not be on PATH in the current terminal session. Ensure %APPDATA%\npm is in your user PATH, or open a new terminal after install, or invoke "%APPDATA%\npm\uipro.cmd" init --ai cursor from the project root.

PowerShell note: Older Windows PowerShell does not treat && as a command separator; run cd and the command on separate lines (see project git_local_push.mdc rule).

UI UX Pro Max vs frontend-design skill
Aspect UI UX Pro Max Frontend-design skill (e.g. Cursor skill)
Strength Searchable datasets (styles, colors, stacks), search.py, optional design-system/MASTER.md + page overrides Execution-focused: distinctive, polished UI; avoids generic “template” aesthetics when building components
Best for Choosing systems, tokens, patterns, stack hints (e.g. nextjs) Craft and visual quality during implementation
💡 Practical combo: use Pro Max to define or refresh constraints and references; use frontend-design when you want strong creative direction in the actual React/Next.js code. Reconcile outputs with existing project rules under .cursor/rules/ (e.g. MOSC styling) so nothing conflicts.
How this integrates with Cursor IDE
  • uipro init --ai cursor installs the skill at .cursor/skills/ui-ux-pro-max/.
  • Cursor loads agent instructions from SKILL.md when that skill is in context.
  • Restart Cursor (or reload the window) after install so skills are picked up reliably.
  • Optional: reference the skill explicitly in chat if your Cursor build supports @-mentions for skills.
Next.js — stack search

Run from the repository root. Adjust the search query to match your feature (e.g. dashboard, auth, marketing).

Stack guidelines
🔧 Customize Parameters:
python .cursor/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --stack nextjs -n 3
Windows single-line:
python .cursor/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --stack nextjs -n 3

Design system (recommended before large UI work)

Design system
🔧 Customize Parameters:
python .cursor/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard fintech" --design-system -p "Roy and Bindu Productions" -f markdown
Windows single-line:
python .cursor/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard fintech" --design-system -p "Roy and Bindu Productions" -f markdown

Persist Master + optional page override

Use --persist (not --pers). This writes design-system/MASTER.md and optionally design-system/pages/<page>.md under the working directory.

Persist design system
🔧 Customize Parameters:
python .cursor/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "Roy and Bindu Productions" --page "dashboard"
Windows single-line:
python .cursor/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "Roy and Bindu Productions" --page "dashboard"

Paths & hierarchy (project-relative):

  • Skill script: .cursor/skills/ui-ux-pro-max/scripts/search.py (not the Claude-only .claude/skills/... path from some blog posts).
  • Hierarchy: for a page like Checkout, check design-system/pages/checkout.md first; if missing, use design-system/MASTER.md only.
  • Align generated guidance with existing .cursor/rules/ (e.g. mosc_styling_standards.mdc, icons_buttons_styles.mdc) so the app stays visually consistent.
