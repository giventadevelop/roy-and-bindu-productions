Purpose of this document

This page explains how the project design system for the Haru Circle — Home 6 Film Online reference was produced using the UI UX Pro Max skill (search.py), where the canonical Markdown files live, and how developers and AI assistants should apply them when implementing pages in Next.js. It follows the project HTML documentation standard (html_documentation_styling_guide.mdc).

How the design system was created
  1. UI UX Pro Max baseline: From the repository root, python .cursor/skills/ui-ux-pro-max/scripts/search.py was run with --design-system --persist, a natural-language query (video / film / streaming / dark cinematic), and -p project name. That produced palette, pattern, and typography suggestions from the skill’s CSV index.
  2. Standard layout: Output was consolidated into design-system/MASTER.md and design-system/pages/home-6-film-online.md. An auto-generated subfolder from the tool was removed so only the canonical paths remain.
  3. Demo fidelity pass: Tokens were merged with the static export demo.harutheme.com/circle/home-6/index.html — notably Nunito Sans (body) and Playfair Display (headings), and surface hex values (#1e1e1e, #161616, #232323, border #333333).
💡 MASTER.md states that Pro Max’s generic font pair (e.g. Outfit / Work Sans) is not used for this reference; the demo export fonts and surfaces override for visual parity.
Files and folder layout
Path Role
design-system/MASTER.md Global source of truth: Pro Max baseline, demo-fidelity typography/surfaces, components, anti-patterns, checklist.
design-system/pages/home-6-film-online.md Page-specific overrides for Home 6 (section rhythm, layout, film cards, WooCommerce note).
.cursor/skills/ui-ux-pro-max/scripts/search.py Skill script used to generate or refresh design-system recommendations.
demo.harutheme.com/circle/home-6/index.html Local reference HTML (WordPress theme export) used to extract real tokens.
Hierarchy (Master vs page overrides)

Rule:

  1. When building a specific page, open design-system/pages/<page>.md if it exists.
  2. If the page file exists, its rules override MASTER.md for that page only.
  3. If there is no page file, use MASTER.md exclusively.
What was merged (Pro Max vs demo HTML)
Aspect UI UX Pro Max (generated) Demo (authoritative for this reference)
Headings / body Suggested pairs (e.g. Outfit / Work Sans) Playfair Display + Nunito Sans
Surfaces Black / navy / rose CTA #1e1e1e, #161616, #232323, borders #333
CTA accent #E11D48 (retained in Master) Use for primary actions unless brand changes
How to use when building UI
  • Implementing Home 6–style screens: Read design-system/pages/home-6-film-online.md, then fill gaps from MASTER.md.
  • Other routes: Use MASTER.md only, or add a new file under design-system/pages/ with only deviations.
  • AI prompts: Point the assistant at both files and ask for components that match tokens (fonts, hex, spacing) and the checklist at the bottom of Master.
  • Do not silently mix unrelated palettes (e.g. MOSC cream theme) on a Home 6 clone unless the product owner explicitly merges brands.
Regenerate or extend (command)

Run from the repository root. Adjust query, project name, and page slug; then merge results into design-system/ as needed.

Persist design system
🔧 Customize Parameters:
python .cursor/skills/ui-ux-pro-max/scripts/search.py "video film streaming dark cinematic landing woocommerce" --design-system --persist -p "Circle Home 6 Film Online" --page "home-6" -f markdown
Windows single-line:
python .cursor/skills/ui-ux-pro-max/scripts/search.py "video film streaming dark cinematic landing woocommerce" --design-system --persist -p "Circle Home 6 Film Online" --page "home-6" -f markdown
⚠️ After re-running: The tool may write a new subfolder under design-system/. Copy merged content into MASTER.md / pages/ manually, or delete duplicates — this project keeps a single canonical design-system/MASTER.md at the repo root.
