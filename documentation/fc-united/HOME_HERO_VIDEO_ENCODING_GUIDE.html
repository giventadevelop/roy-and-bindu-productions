Overview

The homepage hero (/) uses a looping muted MP4 as a full-bleed background (similar to ambient hero patterns on marketing sites), with a dark gradient overlay so headline and countdown text stay readable. The first 39 seconds of the source WhatsApp export are encoded to 1920×1080 H.264, with scale + center crop to reduce letterboxing from mobile-style source frames.

Encoding is performed locally with ffmpeg (not in CI). The repo ships encode-hero-video.ps1 and the React component FcUnitedHeroVideo.tsx that references the generated asset. Output includes a silent AAC stereo track (not video-only) for reliable Chrome playback — see Playback troubleshooting & fixes.

💡 Web playback hints: The player uses autoPlay, muted, playsInline, and preload="auto". Looping is handled in JavaScript (ended → seek to 0 + play()), not only the HTML loop attribute, for Chrome compatibility. A poster image is shown until the video loads or if the file is missing.
⚠️ Prompt-only AI video tools: Tools that only generate video from prompts (e.g. some “AI editor” products) are not a substitute for trimming, cropping, and re-encoding. Use ffmpeg, DaVinci Resolve, Premiere Pro, or HandBrake for precise cuts and web export.
⚙️ Prerequisites
  • ffmpeg on your PATH. Install on Windows e.g. winget install Gyan.FFmpeg or Chocolatey choco install ffmpeg.
  • Source video present at: public/images/md_strikers_media/md_media/WhatsApp Video 2026-03-24 at 10.56.17 AM.mp4
  • Run the encode script from the repository root (or ensure paths resolve).
📋 Encode hero video (PowerShell)

Runs encode-hero-video.ps1: trims to 39 seconds, scales to cover 1920×1080, center-crops, muxes a silent AAC stereo track (via anullsrc), adds faststart for streaming.

Setup — Encode script
🔧 Parameters

No CLI arguments — paths are fixed inside the script. Run from repo root.

powershell -ExecutionPolicy Bypass -File scripts/encode-hero-video.ps1
Windows single-line:
powershell -ExecutionPolicy Bypass -File scripts/encode-hero-video.ps1
✂️ Trim-only (manual reframe)

If you prefer to remove letterboxing in DaVinci Resolve / Premiere first, use this trim-only one-liner (no crop). It uses -an (video-only). For Chrome reliability, prefer the main encode script or remux silent audio into the result. Adjust paths if needed.

Advance — ffmpeg trim-only
🔧 Parameters

Fixed paths in command; run from repository root in a shell where ffmpeg is on PATH.

ffmpeg -y -i "public/images/md_strikers_media/md_media/WhatsApp Video 2026-03-24 at 10.56.17 AM.mp4" -t 39 -c:v libx264 -pix_fmt yuv420p -crf 23 -an -movflags +faststart "public/images/md_strikers_media/md_media/hero-home-ambient.mp4"
Windows single-line:
ffmpeg -y -i "public/images/md_strikers_media/md_media/WhatsApp Video 2026-03-24 at 10.56.17 AM.mp4" -t 39 -c:v libx264 -pix_fmt yuv420p -crf 23 -an -movflags +faststart "public/images/md_strikers_media/md_media/hero-home-ambient.mp4"
⚙️ Script behavior (encode-hero-video.ps1)

What the script does:

  • ffmpeg -t 39 — first 39 seconds only.
  • Video filter: scale=1920:1080:force_original_aspect_ratio=increase,crop=1920:1080 — fill 16:9 then center crop (typical fix for phone video letterbox).
  • -c:v libx264 -pix_fmt yuv420p -crf 23 -preset medium — broad browser support.
  • Audio: -f lavfi -i anullsrc=channel_layout=stereo:sample_rate=44100 with -map 0:v:0 -map 1:a:0, -c:a aac -b:a 96k, -shortest — silent stereo AAC (keeps Chrome from freezing / breaking loop on video-only MP4; in-browser playback stays muted).
  • -movflags +faststart — moov at start for progressive web playback.
📊 Application integration
  • Component: src/components/fc-united/FcUnitedHeroVideo.tsx — client <video> with poster ${FC_IMG}/anim-bg-copyright.jpg, source URL constant HERO_HOME_AMBIENT_MP4 = /images/md_strikers_media/md_media/hero-home-ambient.mp4. Does not rely on the HTML loop attribute alone: ended seeks to 0 and calls play(); a light stuck watchdog recovers if currentTime stops advancing mid-playback. Only calls play() when the element is paused (avoids disrupting Chrome’s decoder by spamming play() on every canplay / canplaythrough).
  • Page: FcUnitedHomePage.tsx — hero section uses <FcUnitedHeroVideo /> under an absolute layer, then a gradient overlay for text contrast.
  • Related: FcEventCountdown.tsx defers live clock to useEffect (null until mount) so SSR and hydration match — avoids a hydration error that could force a client re-render of the page (previously mistaken for video “freezing”).
✅ After encoding: Place hero-home-ambient.mp4 under public/images/md_strikers_media/md_media/ and reload /. If the file is absent, the poster image displays until the asset exists. For Chrome-specific freezes, see section 7.
🔧 Playback troubleshooting & fixes

These issues were hit in production-style testing (Chrome vs Edge, Clerk middleware, React hydration). Use this section when the hero MP4 404s, stalls mid-playback, or fails to loop.

Chrome: freeze mid-playback & broken loop (video-only MP4)

Symptom: Microsoft Edge plays the file smoothly; Google Chrome freezes after ~20–40s and/or does not loop. Cause: Chrome is sensitive to video-only MP4s (ffmpeg -an): the media pipeline can stall and loop behavior is unreliable.

Fix (encoding): Mux a silent AAC stereo track with the same duration as the video (anullsrc + -shortest). The main encode-hero-video.ps1 already does this. The in-page player remains muted; users do not hear audio.

Quick fix: add silent audio to an existing file (no video re-encode)

If you already have a video-only hero-home-ambient.mp4, remux with copy for video and AAC for audio, then replace the file (or adjust paths). From repository root:

Remux — silent AAC (copy video)
🔧 After running

Rename the temp file to replace the original, e.g. PowerShell: Move-Item -Force hero-home-ambient.tmp.mp4 hero-home-ambient.mp4 in the same folder.

ffmpeg -y -i "public/images/md_strikers_media/md_media/hero-home-ambient.mp4" -f lavfi -i anullsrc=channel_layout=stereo:sample_rate=44100 -map 0:v:0 -map 1:a:0 -c:v copy -c:a aac -b:a 96k -shortest -movflags +faststart "public/images/md_strikers_media/md_media/hero-home-ambient.tmp.mp4"
Windows single-line:
ffmpeg -y -i "public/images/md_strikers_media/md_media/hero-home-ambient.mp4" -f lavfi -i anullsrc=channel_layout=stereo:sample_rate=44100 -map 0:v:0 -map 1:a:0 -c:v copy -c:a aac -b:a 96k -shortest -movflags +faststart "public/images/md_strikers_media/md_media/hero-home-ambient.tmp.mp4"

Clerk middleware: MP4 returns 404 HTML instead of video

Symptom: Network tab shows 404 for /images/.../hero-home-ambient.mp4 with headers like x-clerk-auth-reason: protect-rewrite and HTML body instead of binary video.

Cause: Next.js runs Clerk middleware on matched routes. Static extensions must be excluded from the matcher; .mp4 was not originally listed, so Clerk ran auth.protect() on the asset URL.

Fix: src/middleware.ts config.matcher includes video/audio extensions (e.g. mp4, webm, aac) so those requests skip Clerk and Next.js serves public/ files normally.

React: hydration mismatch on countdown (not the video file)

Symptom: Console reports hydration mismatch on FcEventCountdown (server vs client seconds differ).

Fix: Do not initialize countdown state with Date.now() on both server and client. Use null until useEffect runs, then start the interval — see FcEventCountdown.tsx.

⚠️ Avoid: Calling video.play() on every canplay / canplaythrough / loadeddata while the clip is already playing — can contribute to stutter. Prefer play only when paused plus visibility / ended handlers (as in FcUnitedHeroVideo.tsx).
