# TestSprite Report Files Guide

## 📄 Report Files Generated

After running TestSprite tests, the following report files are created:

### 1. **JSON Results** (`tmp/test_results.json`)
- **Format**: JSON
- **Purpose**: Machine-readable test results
- **Updated by**: Filtering script updates this file
- **Use**: For programmatic access to test results

### 2. **Markdown Report** (`tmp/raw_report.md`)
- **Format**: Markdown
- **Purpose**: Human-readable test report (generated by TestSprite)
- **Updated by**: TestSprite MCP (not updated by filtering script)
- **Use**: Quick text-based review of test results

### 3. **HTML Report** (`tmp/test_report.html`) ⭐ **RECOMMENDED**
- **Format**: HTML
- **Purpose**: Beautiful, interactive test report with error details
- **Generated by**: `scripts/generate-testsprite-html-report.js`
- **Updated by**: Automatically generated when filtering script runs
- **Use**:
  - **Open in browser** to review test results visually
  - **Share with Cursor AI** to help fix failures
  - **Print or export** for documentation

## 🚀 How to Use Reports

### Step 1: Run Tests
```bash
node E:\.npm-cache\_npx\8ddf6bea01b2519d\node_modules\@testsprite\testsprite-mcp\dist\index.js generateCodeAndExecute
```

### Step 2: Filter Expected Errors (Optional but Recommended)
```bash
node scripts/filter-testsprite-errors.js
```

This will:
- ✅ Filter expected Next.js 15 warnings from JSON results
- ✅ **Automatically generate HTML report** (`tmp/test_report.html`)
- ✅ Show summary of filtered vs actual failures

### Step 3: Review HTML Report

**Open the HTML report in your browser:**
```bash
# Windows
start testsprite_tests\tmp\test_report.html

# Or double-click the file in File Explorer
```

**The HTML report shows:**
- 📊 Test summary (total, passed, failed, filtered)
- ✅ Passed tests (green)
- ❌ Failed tests (red) with full error details
- ⚠️ Filtered tests (yellow) with explanation
- 📹 Links to test visualization videos
- 🔍 Detailed error messages for debugging

### Step 4: Share with Cursor AI

**To get help fixing failures:**

1. **Open the HTML report** in your browser
2. **Copy the error details** from failed tests
3. **Share with Cursor AI** by:
   - Pasting error messages into chat
   - Or uploading the HTML file
   - Or referencing specific test IDs

**Example prompt for Cursor:**
```
I have test failures in testsprite_tests/tmp/test_report.html.
Test TC001 failed with error: "Social login buttons are missing..."
Please help me fix this issue.
```

## 📋 Report File Locations

All report files are in: `testsprite_tests/tmp/`

```
testsprite_tests/tmp/
├── test_results.json      # JSON results (updated by filter script)
├── raw_report.md          # Markdown report (from TestSprite)
└── test_report.html       # HTML report (generated by filter script) ⭐
```

## 🔍 Understanding Report Contents

### HTML Report Sections

1. **Test Summary**
   - Total tests count
   - Passed/Failed/Filtered counts
   - Success rate percentage

2. **Filtered Tests Info** (if any)
   - Explains why tests were filtered
   - Shows original errors for reference

3. **Test Results**
   - Each test with:
     - Test ID and title
     - Status badge (Passed/Failed/Filtered)
     - Description
     - Error details (if failed)
     - Test visualization link (if available)

### Error Details in HTML Report

Failed tests show:
- **Full error message** with console logs
- **Formatted error text** (code blocks, badges for ERROR/WARNING)
- **Scrollable error content** (for long errors)
- **Test visualization link** (if available from TestSprite)

## 💡 Tips for Using Reports with Cursor AI

### Best Practices

1. **Focus on Failed Tests**: Share only failed test errors, not passed ones
2. **Include Context**: Mention which page/feature the test was testing
3. **Copy Full Error**: Include the complete error message, not just a summary
4. **Reference Test ID**: Use test IDs (e.g., "TC001") for easy reference
5. **Include Screenshots**: If test visualization links are available, mention them

### Example Cursor AI Prompt

```
I have test failures from TestSprite. Here are the details:

Test: TC001 - User Authentication via Social Login
Status: FAILED
Error: Social login buttons are missing or non-functional on the sign-in page...

[Paste full error from HTML report]

The test was trying to click social login buttons on the sign-in page but couldn't find them.
Please help me fix this issue.
```

## 🔄 Regenerating Reports

### Regenerate HTML Report Only
```bash
node scripts/generate-testsprite-html-report.js
```

### Regenerate HTML Report from Custom JSON
```bash
node scripts/generate-testsprite-html-report.js path/to/custom_results.json path/to/output.html
```

### Filter Errors and Generate HTML
```bash
# This automatically generates HTML report
node scripts/filter-testsprite-errors.js
```
